CRITICAL VULNERABILITIES (6)

[CRIT-01] Hardcoded Credentials in Git History
  ├─ File: backend/.env
  ├─ Issue: Database password, JWT secret, SMTP credentials tracked in git
  ├─ Impact: Any developer with git history can access all secrets
  ├─ Fix: git filter-branch, rotate all secrets, add to .gitignore
  └─ Estimate: 30 minutes

[CRIT-02] Instagram OAuth Token in URL Parameter
  ├─ File: backend/auth_service/src/verify.rs:25-28
  ├─ Issue: Access token passed as URL parameter instead of header
  ├─ Impact: Token exposed to logs, browser history, HTTP referrer headers
  ├─ Fix: Use Authorization: Bearer header
  └─ Estimate: 30 minutes

[CRIT-03] Twitter OAuth Token in URL Parameter
  ├─ File: backend/social_service/src/oauth.rs:148-151
  ├─ Issue: Access token in URL query parameter
  ├─ Impact: Same as CRIT-02
  ├─ Fix: Use Authorization: Bearer header
  └─ Estimate: 30 minutes

[CRIT-04] PKCE Challenge Hard-Coded as Plain Text
  ├─ File: backend/social_service/src/oauth.rs:74-91
  ├─ Issue: Challenge = "challenge", verifier = "challenge" (hardcoded)
  ├─ Impact: Anyone can intercept auth code and complete OAuth flow
  ├─ Fix: Generate random S256 challenges, store in session
  └─ Estimate: 3 hours

[CRIT-05] CORS Missing Critical Security Headers
  ├─ File: backend/api_gateway/src/main.rs:121-128
  ├─ Issue: No CSP, HSTS, X-Frame-Options headers
  ├─ Impact: XSS, clickjacking, man-in-the-middle attacks possible
  ├─ Fix: Add SecurityHeaders middleware
  └─ Estimate: 2 hours

[CRIT-06] SQL Injection via String Formatting
  ├─ File: backend/marketplace_service/src/brand_verification_service.rs:177-192
  ├─ Issue: Dynamic SQL column name via format! macro
  ├─ Impact: While restricted now, teaches unsafe patterns
  ├─ Fix: Use SQL CASE expression instead
  └─ Estimate: 1 hour

CRITICAL TOTAL EFFORT: 8 hours

HIGH SEVERITY VULNERABILITIES (5)

[HIGH-01] Weak JWT Validation
  └─ Accepts any algorithm, no explicit validation
  └─ Fix: Specify HS256 algorithm, validate exp/iat/nbf claims
  └─ Estimate: 30 minutes

[HIGH-02] Insecure OAuth State Parameter
  └─ No validation on state parameter length or format
  └─ Could enable CSRF attacks if validation missing
  └─ Fix: Validate state 20-100 chars, alphanumeric only
  └─ Estimate: 1 hour

[HIGH-03] Console Error Logging Exposes Stack Traces
  └─ File: frontend/src/components/ErrorBoundary.tsx
  └─ Stack traces visible to all users in browser console
  └─ Fix: Send to backend observability, show generic message
  └─ Estimate: 2 hours

[HIGH-04] Frontend Generates API Keys
  └─ File: frontend/src/app/brands/onboarding/page.tsx:21
  └─ Keys should be generated server-side only
  └─ Fix: Move key generation to backend
  └─ Estimate: 2 hours

[HIGH-05] Instagram ID as Cryptographic Identifier
  └─ Social IDs are predictable, not cryptographically secure
  └─ Could enable account takeover via ID collision
  └─ Fix: Use UUIDs, verify platform ownership separately
  └─ Estimate: 3 hours

HIGH SEVERITY TOTAL EFFORT: 8.5 hours
